# KubeDeck 首页与 Workloads 页字段级信息架构草案

## 1. 目的

本文档定义首个基础任务流中，首页与 Workloads 页的字段级信息架构。

目标是确保这两页优先服务任务进入与任务完成，而不是服务壳层诊断或框架状态展示。

## 2. 字段优先级模型

本文档使用以下字段优先级：
- `P0`：必须首屏可见，无需额外交互
- `P1`：对任务完成重要，但可以位于首屏主焦点之后
- `P2`：支持性或上下文字段，可折叠、次级展示或默认隐藏

## 3. 首页信息架构

### 3.1 首页首要目标

首页必须帮助用户：
- 确认当前操作上下文，
- 判断下一步最可能要做的任务，
- 并立即进入首条任务流。

### 3.2 首页区块

首页应包含以下区块：
- 全局头部
- 当前上下文区块
- 主任务入口区块
- 最近或默认任务区块
- 阻塞性提示区块

首页不应把诊断信息作为首要区块。

### 3.3 全局头部

字段：
- 产品名称（`P1`）
- 用户身份或会话入口（`P2`）
- 全局设置入口（`P2`）

必须首屏可见：
- 产品名称

可降级展示：
- 设置入口
- 非关键账户细节

### 3.4 当前上下文区块

字段：
- 当前 active cluster（`P0`）
- namespace 范围或上次使用的 namespace（`P1`）
- 当前环境标签（如适用）（`P1`）
- cluster 切换操作（`P0`）

必须首屏可见：
- 当前 active cluster
- cluster 切换操作

可降级展示：
- 次级环境标签
- 扩展 cluster 元数据

### 3.5 主任务入口区块

字段：
- 主目标入口名称，例如 `Workloads`（`P0`）
- 入口用途的简短说明（`P1`）
- 进入动作按钮或可点击卡片（`P0`）
- 可选的默认下一步提示（`P1`）

必须首屏可见：
- 主目标入口名称
- 主进入动作

可降级展示：
- 当空间不足时，简短说明文案可降级

### 3.6 最近或默认任务区块

字段：
- 最近任务或最近页面（`P1`）
- 恢复进入动作（`P1`）
- 默认后备任务（`P1`）

必须首屏可见：
- 如果空间紧张，可以没有

可降级展示：
- 全部最近活动细节

### 3.7 阻塞性提示区块

字段：
- 阻塞问题摘要（`P1`）
- 建议下一步动作（`P1`）
- 关闭或重试操作（`P2`）

必须首屏可见：
- 仅当问题会阻止用户进入任务流时

必须降级或默认隐藏：
- 非阻塞性的壳层诊断
- 原始状态探针
- 后端 endpoint 细节

### 3.8 首页应隐藏或降级的字段

以下内容不应作为首页首屏字段：
- 原始 API target
- registry 类型计数
- 作为主标题信息的 healthz / readyz
- 面向后端壳层检查的失败摘要
- 按 `system`、`user`、`dynamic` 等技术来源分组的导航

## 4. Workloads 页信息架构

### 4.1 Workloads 页首要目标

Workloads 页必须帮助用户：
- 确认当前操作上下文，
- 查看相关资源集合，
- 并快速开始一次标准操作。

### 4.2 Workloads 页区块

Workloads 页应包含以下区块：
- 页面头部
- 上下文与范围栏
- 过滤与搜索栏
- workload 列表区块
- 主操作区块
- 结果或刷新状态区块

### 4.3 页面头部

字段：
- 页面标题，例如 `Workloads`（`P0`）
- 页面用途副标题（`P2`）
- 当前 cluster 标识或标签（`P0`）

必须首屏可见：
- 页面标题
- 当前 cluster

可降级展示：
- 描述性副标题

### 4.4 上下文与范围栏

字段：
- namespace 选择器或范围展示（`P0`）
- 当前资源子类型或 workload 分类（`P1`）
- 当前筛选条件摘要（`P1`）
- 清除筛选操作（`P1`）

必须首屏可见：
- namespace 选择器或范围展示

可降级展示：
- 冗长的筛选说明

### 4.5 过滤与搜索栏

字段：
- 关键词搜索（`P1`）
- 状态筛选（`P1`）
- 资源类型筛选（如需要）（`P2`）
- 排序控件（`P2`）

必须首屏可见：
- 搜索框
- 价值最高的一到两个筛选条件

可降级展示：
- 高级排序控件
- 低频筛选条件

### 4.6 Workload 列表区块

每行或每项字段：
- 资源名称（`P0`）
- 类型（`P1`）
- namespace（如果不是全局固定）（`P1`）
- 状态（`P0`）
- 创建时间或更新时间（`P2`）
- 摘要健康或就绪指示（`P1`）
- 快捷行操作入口（`P2`）

列表中必须首屏可见：
- 资源名称
- 状态
- 能让用户判断当前对象是什么的最小上下文

可降级展示：
- 次级时间信息
- 低频快捷操作

### 4.7 主操作区块

字段：
- `Create` 操作（`P0`）
- `Apply` 操作（`P0`）
- 刷新操作（`P1`）

必须首屏可见：
- 至少一个主操作入口

可降级展示：
- 如果已有自动或内联刷新能力，则刷新操作可以降级

### 4.8 结果或刷新状态区块

字段：
- 最近一次操作结果摘要（`P1`）
- 最近一次刷新状态（`P2`）
- 必要时的重试操作（`P2`）

必须首屏可见：
- 仅当该信息与用户刚完成的操作直接相关时

必须降级展示：
- 与当前 workload 任务无关的持续性壳层诊断信息

## 5. 首屏可见规则

### 首页 P0 字段

以下字段必须在不滚动、不展开次级面板的情况下可见：
- 当前 active cluster
- cluster 切换操作
- 主任务名称
- 主任务进入动作

### Workloads 页 P0 字段

以下字段必须在不滚动、不展开次级面板的情况下可见：
- 页面标题
- 当前 cluster
- namespace 选择器或 namespace 范围
- workload 列表入口区域
- `Create` 或 `Apply` 等主操作入口
- 可见的 workload 名称和状态

## 6. 降级规则

当信息满足以下任一条件时，应被降级或隐藏：
- 它更服务于实现验证，而不是用户动作，
- 它不会影响用户的下一步，
- 它与其他地方已显示的上下文重复，
- 或它暴露的是技术来源分类，而不是任务分类。

## 7. 范围边界

这份信息架构草案暂不定义：
- 精确视觉布局，
- 字体与排版，
- 间距，
- 响应式断点，
- 或组件库选型。

本文档只定义信息应该出现什么、以什么层级出现、以及它们的任务优先级。
