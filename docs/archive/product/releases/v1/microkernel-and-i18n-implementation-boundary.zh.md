# KubeDeck V1 微内核与 i18n 实现边界

## 1. 目的

本文档用于定义：在继续扩大 V1 功能实现前，哪些微内核与 i18n 工作必须先完成。

## 2. 边界原则

目标不是在 V1 之前就把完整插件平台或完整多语言产品全部做完。

目标是先建立最小架构形状与最低 i18n 纪律，避免后续出现结构性回拆。

## 3. V1 必须交付

V1 在微内核与 i18n 前置条件上必须交付：
- 显式的内核贡献合同家族
- 清晰的内核与能力模块归属边界
- 首条内置工作流向能力化归属的映射
- 最低 i18n 运行时规则
- 以及禁止继续无控制扩散硬编码文案

## 4. V1 暂不要求

V1 暂时不需要要求：
- 运行时第三方插件动态加载
- 插件市场界面
- 完整 slot replacement 语义
- 完整语言切换
- 完整翻译覆盖
- 后端 locale 持久化

## 5. 扩展闸门

在以下任一情况成立时，应暂停继续扩大 UI 与工作流实现：
- 内置工作流仍在以壳层特例方式新增
- 后端能力边界仍然是隐式的
- 新增用户文案仍然没有本地化边界

## 6. 完成检查

只有当以下问题都可以回答“是”时，这份边界才算满足：
- 页面/菜单/动作/槽位贡献家族是否已定义？
- 后端能力注册与执行权威是否已定义？
- 内置工作流域是否可以被描述为能力贡献？
- Locale 是否已被视为产品级关注点？
- 是否已有最低规则阻止继续无控制新增硬编码文案？
