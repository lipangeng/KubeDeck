# KubeDeck V1 微内核与 i18n 首补丁检查清单

## 1. 目的

这份清单用于防止第一批微内核/i18n 补丁演变成整个平台级重写。

## 2. 开始前确认

- [ ] 微内核与 i18n 决策文档已被接受。
- [ ] 内核贡献合同草案已被接受。
- [ ] 内核实现映射草案已被接受。
- [ ] i18n 运行时模型草案已被接受。
- [ ] V1 边界文档已被接受。

## 3. 开发中护栏

- [ ] 不引入完整运行时插件加载。
- [ ] 不引入插件市场或管理 UX。
- [ ] 不把后端工作扩大到超出最低合同支持范围。
- [ ] 不尝试完整翻译迁移。
- [ ] 不继续增加新的壳层专属工作流逻辑。

## 4. 必需交付物

- [ ] 内核合同基线
- [ ] 面向内置工作流域的初始 contribution 注册路径
- [ ] 新增 UI 文案的最低文案访问边界

## 5. 偏航信号

如果出现以下情况，说明补丁已偏航：
- [ ] 壳层逻辑继续拥有更多内置工作流行为
- [ ] 贡献合同设计仍然隐含在页面组件内部
- [ ] 新增用户文案仍以任意内联字符串形式加入
- [ ] 完整插件平台范围进入补丁
- [ ] 完整 i18n 铺开进入补丁

## 6. 完成标准

- [ ] 第一补丁建立的是结构，而不是完整扩展范围。
- [ ] 内置工作流已经有清晰路径向 capability-style ownership 演进。
- [ ] 新增 UI 文案不再继续放大未来 i18n 重写成本。
