# KubeDeck V1 微内核与 i18n 首补丁范围

## 1. 目的

本文档定义：引入面向内核的结构与 i18n 最低纪律时，第一补丁的最小范围是什么。

## 2. 补丁原则

第一补丁只建立结构与护栏，不追求完整功能覆盖。

## 3. 第一补丁应包含

第一补丁只应包含：
- 内核贡献类型定义
- 最小 built-in contribution 注册结构
- 最小 shell 到 capability 的组合入口
- 新增 UI 文案的最低文案访问边界
- 在可行情况下覆盖合同稳定性的测试或验证

## 4. 第一补丁不得包含

第一补丁不得包含：
- 完整动态插件加载
- 插件市场或管理 UI
- 大规模后端重写
- 完整运行时语言切换器
- 全量现有页面翻译迁移
- 无关的视觉重设计

## 5. 现有代码规则

第一补丁可以与历史代码共存，但必须停止继续加深“壳层拥有工作流行为”的实现方式。

## 6. 成功标准

当以下条件成立时，第一补丁算成功：
- 内核合同基线已建立
- 内置工作流注册开始向 contribution 边界后移
- 新增 UI 文案已经获得统一的未来可本地化访问路径
